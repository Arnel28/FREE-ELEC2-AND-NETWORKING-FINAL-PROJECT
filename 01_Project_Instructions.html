📌 Project Overview
In this Virtual Performance Task, student groups will use an interactive simulator to demonstrate their ability to install Operating Systems (Windows and Linux), configure a Dual-Boot environment, and establish a functional virtual network. Success is measured by completing all 5 simulation phases and generating a unique Verification Token.
🎯 Learning Objectives
  • 1Demonstrate the Windows 11 installation process in a simulated environment.
  • 2Perform manual disk partitioning (Primary vs Unallocated) within the simulator.
  • 3Demonstrate the Ubuntu Linux installation alongside an existing OS.
  • 4Design and apply a correct subnet scheme within the Virtual Network tool.
  • 5Assign static IPs and verify connectivity using the virtual ping command.
  • 6Generate and submit a Verification Token for final grading.

📋 Virtual Lab Phases & Procedure
1
PHASE 1 – Virtual Windows Installation

Complete the simulated Windows 11 setup with custom partitioning.

  • Select "Custom: Install Windows only (advanced)".
  • CRITICAL: Use the "New" button to create a partition, but leave at least 50GB Unallocated for Linux.
  • Complete the user account (OOBE) setup.
2
PHASE 2 – Virtual Linux Installation

Install Ubuntu using the free space left during the Windows phase.

  • Select "Something Else" for advanced manual partitioning.
  • Create a / (root) partition and swap area using the "New Partition" tool.
  • Wait for the installation progress bar and configure the user profile.
3
PHASE 3 – Subnetting & IP Config

Design a subnet plan for your virtual network and assign static IP addresses.

  • Complete the Subnetting Quiz within the simulator.
  • Assign static IPs to the virtual PCs based on your calculated range.
  • Verification is automatic: the sim will check if your IPs are within the valid subnet.
4
PHASE 4 – Virtual Connectivity Test

Test the virtual network using the built-in ping tool.

  • Open the Virtual Terminal.
  • Ping the other Virtual PC's IP address.
  • Wait for "4 Packets Received" to confirm success.
5
PHASE 5 – Verification Token

Finalize the lab and generate your unique grading code.

  • Once all phases are green, click "Finish Lab".
  • Screenshot the **Verification Token** (12-character code).
  • The token is tied to your Group Name and Section.
