Subnetting — Complete Step-by-Step Guide

For the OS Installation & Network Setup Project — Read this before Phase 4

Subject: Computer Systems Servicing / Networking Topic: IPv4 Subnetting & Static IP Configuration S.Y.: 2025–2026
📌 What is Subnetting?
Subnetting is the process of dividing a large network into smaller, more manageable networks called subnets. In this project, your group will create a small subnet and assign each computer its own unique IP address so they can communicate with each other.
🔢 IP Address
A unique number assigned to each device on a network. Written as 4 groups of numbers separated by dots.
Example: 192.168.10.1
🎭 Subnet Mask
Tells the computer which part of the IP is the network and which part is the host.
Example: 255.255.255.0
🏠 Network Address
The first address in a subnet. This represents the entire network. Cannot be assigned to any device.
Example: 192.168.10.0
📢 Broadcast Address
The last address in a subnet. Used to send messages to all devices. Cannot be assigned to any device.
Example: 192.168.10.255
🧱 Understanding an IP Address

Every IP address has 4 parts (octets) separated by dots. Each part is a number from 0 to 255. Each number can be written in binary (8 bits = 1 octet).

IP Address Part Decimal Value Binary Representation (8 bits)
First Octet 192 110 000 00
Second Octet 168 101 010 00
Third Octet 10 000 010 10
Fourth Octet 0 000 000 00
💡
Tip: You don't need to memorize binary for this project. Just remember: each IP address has 4 numbers (0–255), and the subnet mask tells you how many of those numbers are the "network" part.
🔖 CIDR Notation Explained

CIDR (Classless Inter-Domain Routing) uses a slash number after the IP address to indicate the subnet mask. The number tells you how many bits are used for the network portion.

CIDR Subnet Mask Total Addresses Usable Hosts Use Case in Project
/24255.255.255.0256254Large class (many computers)
/25255.255.255.128128126Medium group
/26255.255.255.1926462Medium group

✅
For this project — use /28 (subnet mask: 255.255.255.240). It gives you 14 usable IPs which is more than enough for 2–3 computers.
🪜 Step-by-Step: How to Calculate Your Subnet

Follow these steps carefully. We will use the example network: 192.168.10.0 /28

1
Choose Your Network Address
Pick the starting IP for your group's network

A network address is just the base or starting address of your subnet. It always ends in .0 (or matches the first address in the range). Your teacher may assign this to you, or you can choose it.

# Common private network ranges you can use: Network A: 192.168.1.0 — most common for home/lab networks Network B: 192.168.10.0 — also common Network C: 10.0.0.0 — enterprise style # For this project, we will use: Network Address: 192.168.10.0 CIDR: /28
⚠️
Write down your chosen network address on your submission paper right after choosing it — you will need it for every calculation below.
2
Find the Subnet Mask from the CIDR Number
Convert /28 into dotted-decimal notation

The CIDR number (like /28) tells you how many bits are "ON" (255) in the subnet mask. Use the cheat sheet from the table above, or use this rule:

# Rule: Subtract CIDR from 256 to find the last octet Formula: 256 - 2^(32 - CIDR) (for the last changed octet) # For /28: Bits used for network: 28 Bits left for hosts: 32 - 28 = 4 bits Block size: 2^4 = 16 Last octet value: 256 - 16 = 240 # Result: Subnet Mask = 255.255.255.240
📋 Quick Reference — CIDR to Subnet Mask

/24 → 255.255.255.0  |  /25 → 255.255.255.128  |  /26 → 255.255.255.192

/27 → 255.255.255.224  |  /28 → 255.255.255.240  |  /29 → 255.255.255.248

3
Calculate the Total Number of IP Addresses
How many addresses does your subnet contain?
# Formula: 2 ^ (number of HOST bits) Host bits = 32 - CIDR Host bits (/28) = 32 - 28 = 4 Total Addresses = 2 ^ 4 = 16 # This means your subnet has 16 total IP addresses # (but 2 of those are reserved — see Step 4 and Step 5)
4
Find the Broadcast Address
The LAST address in your subnet (reserved)

The Broadcast Address is the very last IP in your subnet. To find it, add the total number of addresses to the network address, then subtract 1.

# Formula: Network Address + Total Addresses - 1 Network Address: 192.168.10.0 Total Addresses: 16 Last Octet of Net: 0 Broadcast = 0 + 16 - 1 = 15 # Result: Broadcast Address = 192.168.10.15 ⛔ NEVER assign this address to a computer!
5
Find the Usable Host IP Range
These are the IPs you can actually assign to computers

The usable range is everything between the network address and broadcast address. Skip the first (network) and last (broadcast) addresses:

# Formula: First Usable IP = Network Address + 1 Last Usable IP = Broadcast Address - 1 Total Usable = Total Addresses - 2 # For our /28 example: Network Address: 192.168.10.0 ← RESERVED (network) First Usable IP: 192.168.10.1 ← assign to Computer A Second Usable IP: 192.168.10.2 ← assign to Computer B Third Usable IP: 192.168.10.3 ← assign to Computer C ...and so on until... Last Usable IP: 192.168.10.14 ← last assignable IP Broadcast Address: 192.168.10.15 ← RESERVED (broadcast) Total Usable Hosts: 16 - 2 = 14
✅
Remember: Always skip .0 and .15 (for /28). Only assign IPs from .1 to .14 to your computers.
6
Count the Number of Usable Hosts
How many computers can connect to this subnet?
# Formula: 2 ^ host bits - 2 # (subtract 2 to remove network and broadcast addresses) Host bits: 4 (because /28: 32 - 28 = 4) Hosts: 2^4 - 2 = 16 - 2 = 14 # Your /28 subnet supports up to 14 computers! # More than enough for your group of 2-3 PCs.
7
Assign IPs to Each Computer (Static IP)
Configure each PC with an IP from the usable range

Pick any IP from the usable range and assign one to each computer. Each computer must have a different IP address. See the instructions below for Windows and Linux:

🖥️ Windows — Set Static IP
1
Press Windows Key + R, type ncpa.cpl, press Enter. This opens Network Connections.
2
Right-click your LAN adapter (Ethernet), then click Properties.
3
Double-click "Internet Protocol Version 4 (TCP/IPv4)".
4
Select "Use the following IP address".
5
Enter your values:
IP Address: 192.168.10.1
Subnet Mask: 255.255.255.240
Default Gateway: 192.168.10.1
6
Click OK to save. Then click OK again to close Properties.
7
Open Command Prompt and type ipconfig to confirm your IP is saved correctly.
🐧 Linux — Set Static IP (Terminal)
1
Open a Terminal (right-click desktop → Open Terminal, or search "Terminal").
2
Type this to see your network adapter name:
ip addr show
Look for something like eth0 or enp3s0.
3
Assign a static IP (replace eth0 with your adapter name):
sudo ip addr add 192.168.10.2/28 dev eth0
4
Bring the interface UP:
sudo ip link set eth0 up
5
Verify the IP was applied:
ip addr show eth0
6
GUI Alternative: Click the network icon → Edit Connections → IPv4 → Set to Manual → Fill in IP, Mask, Gateway → Save.
🔍 How to Verify Your IP is Correct
# On Windows — open Command Prompt (cmd) and type: > ipconfig # On Linux — open Terminal and type: $ ip addr show # OR $ ifconfig # You should see your assigned IP address listed # next to your network adapter (Ethernet or enp3s0, etc.)

📡 How to Test with Ping

After setting up your IPs, test connectivity using the ping command. This sends small data packets to the other computer and waits for a reply.

# On Windows — Command Prompt > ping 192.168.10.2 ← from Computer A to Computer B > ping 192.168.10.3 ← from Computer A to Computer C # Expected GOOD result (success): Reply from 192.168.10.2: bytes=32 time<1ms TTL=128 Reply from 192.168.10.2: bytes=32 time<1ms TTL=128 Reply from 192.168.10.2: bytes=32 time<1ms TTL=128 Reply from 192.168.10.2: bytes=32 time<1ms TTL=128 Packets: Sent = 4, Received = 4, Lost = 0 (0% loss) # BAD result (failure — check your IP or cable): Request timed out. Request timed out. # On Linux — Terminal $ ping -c 4 192.168.10.1 ← sends 4 packets, then stops
⚠️
Firewall Issue: If ping fails on Windows, the Windows Firewall might be blocking it. Go to Control Panel → Windows Defender Firewall → Allow an app or feature → check "File and Printer Sharing" — this enables ping to go through.
